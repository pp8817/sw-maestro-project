# 뉴스 브리핑 품질 평가 체크리스트

이 문서는 Daily News Agent 결과물을 개선할 때 같은 기준으로 출력 품질을 확인하기 위한 기준이다.

## 고정 테스트 관심사

아래 관심사로 반복 실행해 개선 전후 결과를 비교한다.

```text
AI 산업 동향
반도체 공급망
스타트업 투자
```

## 실행 조건

- 키워드별 수집 기사 수: 3
- 요약에 사용할 기사 수: 5
- 뉴스 소스: Google News RSS
- Vector DB: Chroma local
- AI 모드: Upstage API key가 있으면 Upstage, 없으면 Demo

## 수집 품질

- 관심사와 직접 관련 없는 뉴스가 과도하게 섞이지 않는가?
- 같은 링크 또는 같은 내용의 중복 기사가 반복되지 않는가?
- 출처, 발행일, 링크가 누락되지 않는가?
- 키워드 3개가 모두 수집 결과에 어느 정도 반영되는가?

## 검색 품질

- 키워드 metadata filter 적용 후에도 결과가 지나치게 좁아지지 않는가?
- 관련도가 높은 기사가 Top-K에 먼저 나타나는가?
- 특정 키워드 기사만 과도하게 선택되지 않는가?
- 결과가 없을 때 전체 similarity search fallback이 동작하는가?

## 브리핑 품질

- 핵심 요약이 기사 목록의 내용만 바탕으로 작성되었는가?
- 기사에 없는 사실이나 과장된 해석이 추가되지 않았는가?
- 주요 뉴스마다 제목, 핵심 내용, 링크가 포함되는가?
- 여러 기사에서 공통적으로 보이는 흐름이 자연스럽게 정리되는가?
- 문장이 어색하거나 맞춤법 오류가 많지 않은가?

## UX 품질

- 버튼 클릭 후 현재 단계가 화면에 표시되는가?
- 수집/저장과 브리핑 생성 단계를 따로 실행할 수 있는가?
- 이미 저장된 기사가 있을 때 embedding을 건너뛰는 정보가 표시되는가?
- 오류가 발생했을 때 사용자가 원인을 이해할 수 있는 메시지가 보이는가?

## 판정 기준

각 관심사별로 아래 기준을 기록한다.

```text
관심사:
실행일:
수집 기사 수:
새로 저장된 기사 수:
이미 저장되어 건너뛴 기사 수:
선별 기사 수:
관련 없는 기사 수:
중복 기사 여부:
브리핑 만족도(1~5):
수정할 점:
```
